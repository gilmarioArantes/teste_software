\begin{frame}{Teste x Debug}
	\begin{itemize}
		\item Testar e Debugar é a mesma coisa?
		\item a Resposta é...
		\item Não.
		\item Teste objetiva revelar a presença de defeitos.
		\item Debug é o processo de encontrar o corrigir os defeitos.
	\end{itemize}
\end{frame}

%-------------------------------------------------------------------------Slide

\begin{frame}{Teste x Homologação}
	\begin{itemize}
		\item Testar e homologar é a mesma coisa?
		\item a Resposta é...
		\item Não.
		\item os objetivos são diferentes.
		\item Teste objetiva revelar a presença de defeitos.
		\item Homologar objetiva aferir que o funcionamento esteja adequado às necessidades explícitadas.
	\end{itemize}
\end{frame}

\begin{frame}{Tipos de Teste}
  \begin{itemize}
   \item Diferentes tipos de testes podem ser utilizados para verificar se um programa se comporta como o especificado.
   \item Basicamente, os testes podem ser classificados em:
   \begin{itemize}
    \item Teste Funcional (\textbf{teste caixa-preta});
    \item Teste Estrutural (\textbf{teste caixa-branca}) e 
    \item Teste Baseado em Defeitos (\textbf{Testes de Mutantes}).
   \end{itemize}
  \end{itemize}
\end{frame}

%-------------------------------------------------------------------------Slide

\begin{frame}{Tipos de Teste}
  \begin{itemize}
   \item A t\'ecnica de teste \'e definida pelo tipo de informa\c{c}\~ao utilizada para realizar o teste.
  \end{itemize}
\end{frame}

%-------------------------------------------------------------------------Slide

\begin{frame}{Técnica de Teste Funcional}
  \begin{itemize}
    \item \textbf{T\'ecnica caixa-preta} - os testes s\~ao baseados exclusivamente na especifica\c{c}\~ao de requisitos do programa. Nenhum conhecimento de como o programa est\'a implementado \'e requerido.
  \end{itemize}
  \begin{figure}
   \centering
   \includegraphics[width=5cm]{testeCaixaPreta}
   \caption[numbered]\insertcaptionnumber{ - Técnica de Teste Funcional. \cite{neto2009}}
  \end{figure}
\end{frame}

%-------------------------------------------------------------------------Slide

\begin{frame}{Técnica de Teste Estrutural}
  \begin{itemize}
    \item \textbf{T\'ecnica caixa-branca} - os testes s\~ao baseados na estrutura interna do programa, ou seja, na implementa\c{c}\~ao do mesmo.
  \end{itemize}
  \begin{figure}
   \centering
   \includegraphics[width=5cm]{testeCaixaBranca}

    \item \textbf{T\'ecnica baseada em defeitos} - os testes s\~ao baseados em informa\c{c}\~oes hist\'oricas sobre defeitos cometidos frequuentemente durante o processo de desenvolvimento de software.

 \begin{block}{Teste de Regressão}
  \begin{enumerate}
   \item Os testes de regressão geralmente são executados após a correção de algum defeito ou após a adição de uma nova funcionalidade. 
   \item Seu objetivo é garantir que nenhum defeito foi acrescentado ao sistema após sua modificação. 
   \item De nada adianta testar um sistema, verificar que ele não possui defeitos para aquele conjunto de casos de teste e após modificações no sistema, aqueles casos de teste não serem novamente executados, pois as novas mudanças podem trazer defeitos para o sistema.
  \end{enumerate}
 \end{block}
